# Стили проекта
#
#
## Стили
Не использовать свойства шрифтов, а также значения в пикселях и процентах для размеров, разрешается только 100%, также не забывать про классы w-100 h-100,
если вдруг по какой-либо причине это необходимо, комментарий обязателен



## Определение размера экрана
не использовать в компонентах window event resize для этого есть глобальная переменая `$screen`

`$screen.width` - размер экрана

`$screen.sizes` - текущий экран sm md lg xl

`$screen.media` - все медиа условия включающие текущий размер

```
v-if="['sm', 'md'].includes($screen.sizes)"
v-if="$screen.sizes === 'lg'"
```

## Медиа условия
использовать только миксины
`@include media({size}) {}`
{size} - xl, lg, md, sm

если не получается писать коммент

## Спец Классы
`.s-container` - контейнер


## Шрифты
Не использовать миксины, писать классы
`.h1 bold .h2 .h3 .h4 .text .text-sm`


## Иконки
для иконок использовать классы - `icon-{size}`

в имени иконок обязательно в суффиксе указывать размер из дизайна


## Классы
стараться описывать все максимально классами не используя переменных и миксинов scss

`class="d-f m-4 m-md-4"` - норм

`class="d-f m-4 m-md-4 d-f m-4 m-md-4 d-f m-4 m-md-4 d-f m-4 m-md-4"` - плохо, в таких случаях создать класс и описать в нем

`class="d-f custom-class"` - не запрещено, но иногда удобней и предпочтительней написать в классе `display: flex`


### цвета
для `background-color` использовать bg-`name`
```
.bg-primary .bg-danger
```

для `color` использовать color-`name`
```
.color-primary .color-danger
```

### отступы
для `margin` использовать m`{-media}?`-`{size}` (x,y) - оси
```
.m-1 .ml-2 mt-md-1 .mx-4 .my-2
```

для `padding` использовать p`{-media}?`-`{size}` (x,y) - оси
```
.p-1 .pl-2 pt-md-1 .px-4 .py-2
```

### утилиты
для `display-flex: flex` использовать `d-f`

для `align-items: center` использовать `ai-c`

список постоянно пополняется посмотреть можно `theme/styles/main.scss`

<span className={'color-danger'}>создание новых утилит</span>

если свойство состоит из 2 слов использовать 1 буквы слов  `text-align` = `ta`

если свойство состоит из 1 буквы и не конфликтует с другими  `display` = `d`

при конфликте первую и последнюю букву `border` = `br`  или часть слова по логике `background` = `bg`

### Z-index
Z-index подключается через `@include z-index(наименование ключа)`, список ключей можно посмотреть в `theme/styles/_variables.scss`

Значения z-index заданы с шагом 10 для того, чтобы оставить место для ошибок и настроек

При добавлении нового ключа необходимо предусмотреть все конфликты с существующей картой z-index

### Универсальные классы
.link - стилизация ссылки
.link--underline - модификатор добавляет подчеркивание
